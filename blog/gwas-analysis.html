GWAS 分析入门：从群体设计到候选基因解析

全基因组关联分析（Genome-Wide Association Study, GWAS）是连接自然变异、复杂性状和候选基因的重要方法。对作物研究而言，GWAS 不只是画出 Manhattan plot，更关键的是把群体、表型、基因型、统计模型和生物学证据串成一条可解释的分析链。

作者：Jiwen Zhao 主题：GWAS / 作物遗传 / 生物信息学 更新：2026-06-12

1. GWAS 解决什么问题？

GWAS 的核心目标是寻找基因组变异与表型差异之间的统计关联。与双亲群体定位相比，自然群体或育种群体包含更丰富的历史重组事件，因此定位分辨率通常更高，适合挖掘控制株高、产量、抗病性、品质和环境适应性等复杂性状的关键位点。

但是，关联不等于因果。一个显著 SNP 可能只是与真正功能变异连锁，甚至可能受到群体结构、亲缘关系、批次效应或表型误差影响。因此，一次可靠的 GWAS 应该同时回答三个问题：统计信号是否可信、信号区域是否具有遗传学解释、候选基因是否有功能证据支持。

2. 分析前的关键准备

群体设计

样本应尽量覆盖目标研究对象的遗传多样性，同时避免过强的亚群偏倚。作物中常见材料包括地方品种、育成品种、核心种质、重测序群体和区域育种群体。

表型质量

表型是 GWAS 的上限。多环境、多重复和统一评分标准可以降低噪声。对抗病性状，应特别注意病圃压力、发病时期、环境互作和离群样本。

基因型数据

SNP 芯片、GBS、外显子捕获和全基因组重测序都可用于 GWAS。不同平台的标记密度和缺失模式不同，会直接影响定位精度和候选区域大小。

3. 基因型质控：不要跳过的步骤

质控的目的是减少低质量标记带来的假阳性和模型不稳定。常用过滤指标包括缺失率、最小等位基因频率（MAF）、杂合率、样本缺失率和重复样本一致性。对自交作物或高度纯合材料，还需要关注异常杂合位点；对多倍体作物，则要确认变异检测和剂量信息是否适合后续模型。

# 常见质控思路示例
plink --vcf input.vcf \
  --geno 0.1 \
  --mind 0.1 \
  --maf 0.05 \
  --make-bed \
  --out gwas_qc

过滤阈值没有放之四海而皆准的标准。群体规模较小或目标是稀有变异时，MAF 阈值需要谨慎调整；群体结构复杂时，质控后还应重新检查 PCA 和亲缘关系矩阵。

4. 模型选择：校正结构与亲缘关系

GWAS 中最常见的混杂因素是群体结构和个体间亲缘关系。如果不校正，某些群体特异的等位基因会被误判为性状相关位点。常用策略包括在模型中加入 PCA/Q 矩阵作为固定效应，并使用 Kinship 矩阵作为随机效应。

模型 适用场景 注意点
GLM 快速初筛，群体结构较弱 容易出现假阳性
MLM 需要同时校正 PCA 和 Kinship 计算较慢，可能过度校正
FarmCPU / BLINK 大规模标记和复杂性状 需要结合 QQ plot 和重复验证判断稳定性

5. 结果解读：显著峰之后还要做什么？

Manhattan plot 可以展示全基因组关联峰，QQ plot 可以帮助判断整体假阳性膨胀。拿到显著 SNP 后，建议继续做以下解析：

  1. 确定显著阈值，例如 Bonferroni 校正、FDR 或基于置换检验的经验阈值。
  2. 根据连锁不平衡衰减距离定义候选区间，而不是只看单个 SNP。
  3. 检查候选区间内的基因注释、结构变异、非同义突变、启动子变异和表达差异。
  4. 结合已发表 QTL、抗病基因、同源基因、转录组和单倍型证据进行优先级排序。
  5. 在独立群体、不同环境或功能实验中验证关键位点。

6. 作物抗病性 GWAS 的实践建议

抗病性状往往受病原小种、环境条件和发育时期影响。对于小麦条锈病、叶锈病等病害，单环境显著信号可能只反映特定病圃或特定小种下的抗性表现。更稳健的做法是将多环境 BLUP、单环境表型、病情指数和抗感分类结果分别分析，再比较稳定重复出现的位点。

候选基因层面，NLR、RLK、RLP、转录因子、激素通路和细胞壁相关基因都值得关注。但不要只因注释像抗病基因就直接认定候选基因。更可靠的证据链应包括：显著位点、LD 区间、功能变异、表达响应、单倍型效应和已有文献支持。

7. 一个推荐的 GWAS 工作流

表型整理与离群检测 基因型质控与缺失填补 PCA、群体结构和 Kinship 评估 多模型关联分析 Manhattan / QQ plot 可视化 LD 区间与候选基因注释 单倍型、表达和文献证据整合 独立群体验证或功能验证

结语

好的 GWAS 分析不是堆软件参数，而是围绕一个清晰的遗传问题建立证据链。对于作物遗传研究者来说，统计显著性只是起点，真正有价值的是找到可以解释表型、服务育种并能够被进一步验证的候选位点和候选基因。

如果把 GWAS 作为一个完整项目来设计，建议从一开始就同步规划表型采集、样本组成、测序深度、数据质控、统计模型、候选基因注释和后续验证方案。这样最终得到的结果才更容易从图表走向生物学解释。
